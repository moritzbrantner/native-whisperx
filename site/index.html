Workflow Composition

What this repository owns

The repo owns the application path that turns media or samples into WhisperX-compatible outputs. It does not own the lower-level ASR, transcript, model-runtime, alignment, or speaker primitives.

01 Media input

Files, WAV samples, or decoded media.

02 ASR and VAD

Native providers or delegated compatibility paths.

03 Alignment

wav2vec2 CTC alignment unless disabled.

04 Diarization

Optional speaker labels and profile-oriented workflows.

05 Outputs

WhisperX JSON, Native JSON, SRT, WebVTT, TXT, TSV, or AUD.

Parity vocabulary

The words contributors should use

These terms come from the repository domain context and keep discussion precise when work moves between compatibility and native implementation.

WhisperX Parity

Feature compatibility with the Python WhisperX user-facing surface.

Rust-Native Parity

A stricter track where user-visible WhisperX behavior is implemented in Rust/native repository code.

Native

The Rust-first direction of the project, not a claim that every feature is Rust-only today.

Delegated Feature

A feature satisfied by Python WhisperX or a Python dependency while a Rust path matures.

Parity Harness

The fixtures, runner, structured diff, and reports used to compare behavior.

WhisperX JSON

The default user-facing JSON contract compatible with Python WhisperX.
